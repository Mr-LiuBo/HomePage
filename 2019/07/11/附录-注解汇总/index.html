附录--注解汇总

发表于 2019-07-11 分类于 Java ， Spring

Hibernate相关注解

@Column

@Column(name=”COLUMN_NAME”,unique=true)

常用属性：

name：列名
unique：是否唯一
nullable：是否可空
length：对于字符指定最大长度
insertable：是否允许插入
updatetable：是否允许更新
columnDefiniton：建表时创建此列DDL
secondaryTable:从表名

@DynamicInsert、@DynamicUpdate

  • hibernate和JPA的注解，注解在实体类上
  • 动态插入和更新
  • 默认是false，当设置为true时，如果该字段的值为null，则不会加入到inset和update语句中

@Entity

  • hibernate中该注解说明这是一个实体类，并且使用默认的orm规则。即class名即数据库表名，class字段名即表彰的字段名。如果想改变这种ORM规则。要使用@Table和@Column

@GeneratorValue、@GenercGenerator

@GeneratorValue

JPA通用策略生成器属性：generator：主键生成器分配一个名称。

@GenercGenerator

自定义主键生成策略，具有如下属性

name：生成器名称
strategy：生成器类名
parameters：参数

@Id

  • hibernate中该注解，用于成员变量，说明这是一个主键

@Tbale

  • 用于hibernate中注解，@Tbale(name=”TABLE_NAME”)

Lombok

@Data、@Get、@Set

  • 实体类省略get/set、toString方法，pom.xml引入依赖 。

@Slf4j

  • 日志logback，需要引入lombok

MyBatis

@Parm

  • 用于注解dao层的方法入参的参数名，简化xml配置

  • MyBatis（DAO -> XML）传参

数据显示格式化

@JsonProperty

  • 指定返回的json数据的key

@RequestParam

  • @RequestParam(name=”id”,required=false,defaultValue=”0”) String deteId
  • name指定传参名称（value与name作用完全相同，源码中互相为别名）
  • required=false 可空 ，默认是不为空
  • defaultValue 缺省时默认值
  • get请求传List等参数类型时也需要添加该注解，并指定name/value

@ResquestBody

  • 读取Resquest请求的body部分数据，使用系统默认的HttpMseeageConverter进行解析，然后把相应的数据绑定要返回的对象上，再把HttpMseeageConverter返回的对象数据绑定到Controller中方法的参数上

@ResponseBody

  • 将Controller的方法返回的对象，通过适当的HttpMessageConverter转换为制定格式后，写入到Response对象的body数据区，使用情况：返回的数据不是Html标签的页面，而是其他数据格式的数据时，（如Json、xml等）使用；

持久层查询时间返回的是long值

实体类根据json包添加响应注解

依赖注入

@Autowired

@Autowire注解按照类型，即注解的字段的类型寻找该类型的实例bean，这种方式成为byType。（先按byType方式，没找到通过byName方式，再失败后报错）。通常与@Qualifier配合使用。

  • 用于成员变量。

  • 适用方法。

@ImportResource

  • 用于配置类，可以使用@ImportResource从一个或多个XML文件中导入bean的定义

@Qualifier

@Qualifier("XXX") Spring的Bean注入配置注解，该注解指定注入的Bean的名称。

@Resource

@Resource是JDK提供的注解，默认就是按照byName的方式寻找bean，一般一个name对应一个bean，当找不到与名称匹配的bean才会按照类型装配（byType），再找不到报错。

@Scope

配合@Component或者@Bean一起使用。用于声明该Bean的作用域。

针对会话、请求情况，必须指定proxyMode属性 如：
@Scope(value = “session”,proxyMode = ScopedProxyMode.TARGET_CLASS)

注入的是类，proxyMode 可以是 ScopedProxyMode.TARGET_CLASS；

接口的情况下，取值是 ScopedProxyMode.INTERFACES

详细示例可参考：SpringBean定义的修改作用域部分

配置相关

@ComponentScan

默认规则下，它会扫描该注解所在类同包目录下组件。指定扫描包有以下方式。

  • @ComponentScan("packageName")
    或 @ComponentScan(basePackages={"package1","package2"})
  • @ComponentScan(basePackageClasses={ClassName1.class,ClassName2.class}

@Configuration

  • 作用于类，此注解标明这是一个Spring 的配置类，在项目启动的时候，会自动扫描此配置；

@ConfigurationProperties

@PropertySource

  • 用于类，这个注解是读取properties文件 不需要我们自己写读取properties的文件的代码，SpringBoot帮我们完成了，十分方便；classpath一定不能丢；
